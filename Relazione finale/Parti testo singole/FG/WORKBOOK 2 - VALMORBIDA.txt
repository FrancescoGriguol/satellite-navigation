WORKBOOK 2 - VALMORBIDA

L'esercitazione aveva l'obiettivo di computare le posizioni dei satelliti e gli errori sui loro clock usando due diverse tipologie di dati:
a. precise product scaricati dal sito web IGS (% magari mettere il link ) 
b. dati trasmessi dal broadcast navigation message
Le posizioni ottenute sono state quindi usate per visualizzzare gli archi di orbita percorsi durante il tempo di simulazione in uno scenario 3D.
Il focus dell'esercitazione si spostava poi sulla coparazione dei risultati delle due diverse tipologie di dati elaborati, in modo da riscontrare la divergenza dei dati dopo un tempo di interpolazione di circa 2h, come previsto dalla teoria. 
In conclusione poi era richiesto lo skyplot dei satelliti e il plot degli elementi dell'orbita osculatrice.

Lo svolgimento di ogni punto ha previsto l'uso esclusivo di MatLab per la generazione di script e function che leggessero i dati dai file sp3 e RINEX ed elaborassero i dati.
I tre script principali al loro interno richiamano alcune funzioni già presenti in matlab per operazioni di manipolazione dei dati che sono di scarso interesse al fine di questa esercitazione e pertanto la trattazione della metodologia di analisi sarà concentrata principalmente sui codici attivamente modificati.

1. Broadcat_orbits:
	
	Le prie righe di codice hanno l'importante compito di settare le impostazioni iniziali dello script, in particolare si specificano:
- il file RINEx dal quale ottenere i dati
- la costellazione di satelliti da analizzare
- gli ID dei satelliti che si vogliono studiare
- il tempo di interpolazione
Successivamente si comanda la lettura dei dati delle effemeridi e vengono recuperate informazioni aggiuntive sul file RINEX ( rinexread - rinexinfo ) e si estraggono i dati delle effemeridi per la costellazione selezionata e i dati specifici delle effemeridfi per i satelliti selezionati.
Successivamente si inizializza un ciclo for per il calcolo delle coordinate x y z delle posizioni orbitali a partire dai dati delle effemeridi. Questa operazione viene fatta dopo aver discriminato la costellazione ( se GPS o GLONASS ) in modo che all'interno del ciclo for venga chiamata la rispettiva function (GPS_coordinates o GLONASS_coordinates).

Una volta che sono state memorizzate le posizioni è stato usato satellite.Scenario per la visualizzazione 3D delle orbite.

2. Precise_Orbits

	Analogamente a quanto detto prima il codice setta le impostazioni iniziali dello script, in particolare si specificano:
- il file SP3 dal quale ottenere i dati
- la costellazione di satelliti da analizzare
- gli ID dei satelliti che si vogliono studiare
- il tempo di interpolazione
le prime operazioni che vengono eseguite dallo script sono quelle di lettura del file dati per la costellazione specificata e l'estrazione dei satelliti dai dato del file di precision positioning. viene poi richiamato Satellite.Scenario per la visualizzazione 3D delle orbite.
Lo step successivo è quello di ottenere i dati interpolati, attraverso la function interp_precise_sp3, di cui si fornisce un'overview in seguito, per poi confrontare in un grafico i dati sp3 e i dati interpolati per la componente x del satellite.
(%MAGARI METTIAMO COME SUBSECTION LA FUNCTION PER FARE L'INTERPOLAZIONE, COME HO SCRITTO SOPRA)
(% RIASSUNTO:Questo codice MATLAB elabora, interpola e visualizza i dati delle orbite satellitari dai file SP3 per la costellazione GPS. Legge i dati delle effemeridi, estrae le posizioni dei satelliti, calcola le orbite interpolate e visualizza le orbite in uno scenario 3D. Infine, confronta i dati delle effemeridi con i dati interpolati, permettendo di verificare la precisione dell'interpolazione.)

3. Compare_broadcast_precise_orbits


	La struttura dello script presenta inizialmente le stesse caratteristiche di entrambi gli script scritti in precedenza, necessarie per ottenere i dati sul quali si vuole fare il confronto. Una volta ottenute le posizioni ricavate dalle effemeridi broadcast e i dati interpolati per il precise message si calcolano le differenze tra i due per tutte e tre le coordinate spaziali, che poi vengono plottate in un grafico in funzione del tempo di interpolazione.
